栈

  栈和队列是线性表的特殊情况，是限制存取位置的线性表。线性表、栈和队列的逻辑结构都是线性的，有唯一的前驱后继，且是一对一的关系，但是只有线性表是可以在任意位置插入和删除元素的线行结构。而栈只能在栈顶位置进行插入和删除元素，队列只能在队头删除元素，在队尾插入元素。所以栈和队列被称特殊的线性表。

栈是一个重要的数据结构，应用十分广泛。栈是只允许在表的末端进行插入和删除，也称为栈顶；在另一端则称为栈底。在栈中，如果元素按照 A、B、C、D有序顺序入栈的话，那么出栈的顺序只能是D、C、B、A，秉承着先进后出，后进先出的原则。下面2张图表明了栈的入栈、出栈的过程：

栈的基本操作

那么，在一个栈中应该有的基本操作：

  • 初始化：建立一个空栈
  • 取栈顶元素：当栈不为空时，则取栈顶的元素返回成功状态；否则返回不成功
  • 入栈：当栈不满时，将栈顶插入一个新元素，返回成功信息；否则返回不成功
  • 出栈：当栈不为空时，删除栈顶元素，返回成功信息；否则返回不成功
  • 判断栈是否为空：如果栈为空返回TRUE；否则返回FALSE
  • 遍历整个栈：遍历整个栈的元素

class SeqStack{
private:
    int maxLength;
    int top;
    T *data;

template<class T>
Status SeqStack<T>::Push(T elem){
    //判断栈是否已满
    //条件是top指针是否等于maxLength-1
    if(top==maxLength-1)
        return RANGE_ERROR;
    else{
        data[++top]=elem;

Status SeqStack::Pop(){
    //判断栈是否是空的

        return EMPTY;
    else{
        top--;
        return SUCCESS;
    }

}

template<class T>
int LinkStack<T>::getLength()const{
    //判断栈是否为空
    if(top==NULL)
        return -1;
    else{
        int count=1;
        while(top!=NULL){
            count++;
            top=top->next;
        }
    }
    return count;
}

    //判断栈是否为空

        return EMPTY;
    else
        elem=top->data;
    return SUCCESS;
}

入栈

template<class T>
Status LinkStack<T>::Push(T elem){
    Node<T>* p = new Node<T>(elem,top);
    top=p;
    return SUCCESS;
}

出栈

template<class T>
Status LinkStack<T>::Pop(){
    //判断栈是否为空
    if(top==NULL)
        return EMPTY;
    else{
        Node<T>* p=top;
        top=top->next;
        delete p;
        return SUCCESS;
    }
}

栈的应用——表达式求值

计算机进行算数运算通常是用栈来实现的，算术运算终点讨论后缀表达式和中缀表达式。

  • 中缀表达式
    这种表达式表示运算符总是置于与之相关的2个运算对象之间，这种表达法被称为中缀表达式

             1+2(3-45)/6

  • 后缀表达式
      这种表达方式的一个特点：表达式中各个运算时按照运算符出现的顺序进行的，所以不用括号来指示运算顺序，跟上面的式子等价的后缀表达式是：
             1 2 3 4 5 * - * 6 / +

  后缀表达式的运算规则：读数字则继续往下读，读到运算符停止，在这个运算符之前的2个操作数做运算得到一个结果，然后继续往下读字符。

那么上述表达式的运算顺序是：4 * 5的结果是20，在往下走是 - ，该操作符上面的2个操作对象，一个是20，一个就是3，3 - 20的结果是 -17；那么继续往下读字符，读到 * ，*号的操作对象有-17和2，2 * - 17 的运算结果是 -34，继续往下读到数字6，不做理会继续往下读，都到运算符/，/运算符之前的操作对象有 -34 和6，-34/6的结果约等于 -5.7，继续往下读字符，读到+运算符，+运算符的操作对象有-5.7和1，-5.7+1= -4.7。这个是最后的结果，与中缀表达式的结果一样。但是这种计算的实现相对于计算机要简单的多。

队列

  队列和栈相反，队列是一种先进先出的线性表，它只允许在表的一端插入元素，而在另一端删除元素。在队列中，允许插入的一段叫做队尾（rear），允许删除的一段叫做队头（front）。与我们日常生活中的排队是同一个现象。

队列的基本操作

  • 初始化：初始化一个空队列
  • 求长度：统计队列中数据元素的数列
  • 取队头元素：当队列不空时，取队头的数据，返回成功状态；否则返回不成功状态。
  • 遍历队列：遍历输出队列里的元素
  • 判断队列是否为空：如果队列为空，返回TRUE；否则返回FALSE；
  • 入队：当队列不满时，在队尾插入一个新元素，返回成功信息；否则返回不成功状态。
  • 出队：当队列不为空时，删除队头元素，并返回成功信息；否则返回出队失败状态。
  • 清空队列：将队列设置为空队列。

队列的顺序存储实现方式

  队列需要设置两个指针，分别是队头指针（front）和队尾指针（rear），指向队列的队头元素和队尾元素。在这里规定：队头指针front指向队头元素的所在位置；队尾指针（rear）指向队尾元素的后一个位置（像STL中的end迭代器一样）。注意，这里的出队其实在顺序存储实现方式中，也就是数组的实现方面其实并不是删除元素，而是将队头指针往后挪，而那个元素已经不要了，可以被新的元素替代。

举个例子，假设有4个存储空间的队列，队列的最大长度maxSize为4，那么队列最多允许存储4个元素。初始化队列为空，接着A、B、C、D依次入队，然后A、B出队，这是队头front指针指向C所在位置，队尾指针rear指向第4个元素D，下标是3。如果元素E要入队，队尾指针要+1，这是队尾指针已经是3且maxSize是4，再加1要溢出了，但是队列已经出队了2个元素，此时队列还有2个存储空间也就是A和B的存储空间可以存储，这种不是因为存储空间不够而产生的溢出，称为假溢出。针对这个情况使用循环队列，就是把顺序队列所使用的存储空间构造成一个逻辑上首尾项链的循环队列，称为循环队列。

解决思路：当队尾指针rear或者队头指针front指向maxSize-1的位置的时候,如果再进一个位置，就自动到0下标处，这一步可以通过求模（取余%）实现，取余的值就是maxSize。例如maxSize=4，rear=3，再加一rear=（rear+1）%4=0，队尾下一个位置就变成了0，那么就可以实现这个循环数组了。

但是问题来了，如果按照上述方法实现的话，队列空和队列满这2中情况就相同无法分辨了。假设一个队列maxSize为5，A、B、C、D、E元素依次入队，下标分别是0、1、2、3、4，队尾指针rear根据rear=（4+1）%5=0，指针则会指向下标0位置，和队头指针front所指同一个元素。我们都知道，队列空时，队头指针front和队尾指针rear都是指向同一个位置下标[ 0 ]，那么该怎么区分这2种情况呢？对这2种情况的判断条件分别为:

  • 当队列为空时：
    front==rear;
  • 当队列满时：
    front==(rear+1) % maxSize;

顺序队列实现

顺序队列类

template<class T>
class SeqQueue {
private:
    T* data;            //数组
    int front, rear;    //队头队尾
    int maxSize;        //该数组的最大容量

public:
    //该类的方法应该包含： 初始化空队列、销毁队列、入栈、出栈、获取队头元素、display函数、获取长度等其他的函数
    SeqQueue(int size);
    ~SeqQueue();
    Status getFront(T& elem)const;
    //Status getLength(T& length)const;
    bool empty()const;
    void display()const;
    Status Push(T elem);
    Status Pop();
};

构造函数

//构造函数
template<class T>
SeqQueue<T>::SeqQueue(int size) {
    data = new T[size];
    maxSize = size;
    front = rear = 0;
}

//析构函数
template<class T>
SeqQueue<T>::~SeqQueue() {

获取队头元素

//获取队头元素
template<class T>
Status SeqQueue<T>::getFront(T& elem)const {
    //先判断队列是否为空
    if (front == rear)
        return EMPTY;
    else 
        elem = data[front];
    return SUCCESS;
}

遍历队列函数

//遍历队列函数
template<class T>
void SeqQueue<T>::display()const {
        for (int i = front;i != rear;i = (i + 1) % maxSize) 
            cout << data[i];
}

判断队列是否为空

//判断队列是否为空
template<class T>
bool SeqQueue<T>::empty()const {
    return (front == rear)? true:false;
}

入队

//入队
template<class T>
Status SeqQueue<T>::Push(T elem) {
    //判断队是否满了
    //队尾+1的和取最大元素的余数==队头的话，表示队列已满
    if ((rear + 1) % maxSize == front)
        return OVER_FLOW;
    else {
        data[rear] = elem;
        rear = (rear + 1) & maxSize;
        return SUCCESS;
    }
}

出队

//出队
template<class T>
Status SeqQueue<T>::Pop() {
    //判断队是否是空的
    if (front == rear)
        return EMPTY;
    else {
        front = (front + 1) % maxSize;
        return SUCCESS;
    }
}

//清空链队
template<class T>
void LinkQueue<T>::clear() {
    Node<T>* p = front;
    while (p!=NULL) {
        front = p->next;
        delete p;
        p = front;
    }
    rear = front;
}

入队

//入队
template<class T>
Status LinkQueue<T>::Push(T elem) {
    //如果系统空间不够则返回溢出
    //这里进队有2个状况，一种是空的队列要元素入队，一种是非空队列入队
    //空队列的情况下，front和rear都是NULL，这时候元素入队，front和rear都会指向这个元素
    //如果是非空队列的情况下入队，front指针不用做修改，而rear会指向这个新元素
    //所以在函数体里面需要考虑这2种情况，要做一个判断，判断rear==NULL，如果等于NULL，那么表示是空队列，front和rear都要指向新元素
    Node<T>* p = new Node<T>();
    p->data = elem;
    if (p)
    {
        if (rear!=NULL)
        {
            rear->next = p;
            rear = rear->next;
        }
        else
        {
            front = p;
            rear = p;
        }
        return SUCCESS;
    }
    else
    {
        //如果p为NULL，那么就是内存空间不足，创建不了结点
        return OVER_FLOW;
    }
}

出队

//出队
template<class T>
Status LinkQueue<T>::Pop() {
    //先判断这个队列是不是空的，如果是空的直接结束程序
    //如果队列是非空队列，那么就做出队操作
    //注意，出队之后，front需要向后移动的，如果要出队的结点是最后一个结点的时候，front向后移就是NULL了，rear指针也会因为元素的出队而指向一个不存在的结点。
    //这时需要做一个判断，在front做移动之后，指向NULL的时候，那么rear也要等于NULL

    Node<T>* p;
    //判断队列是否为空
    if (front != NULL) 
    {
        p = front;
        front = front->next;
        //判断front移动之后队列是否变成空队列了
        //如果是空队列，rear==null
        if (front==NULL)
            rear = NULL;
        delete p;
        return SUCCESS;
    }
    else
        return EMPTY;
}
