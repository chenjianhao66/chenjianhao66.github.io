一、Spring简介

1.1 Spring体系架构

Spring是一个轻量级的 Java 开发框架，为了解决企业级应用开发的业务逻辑层和其他各层的耦合问题。

1.2 核心容器

Spring的核心容器是其他模块建立的基础，由Spring-core、Spring-beans、Spring-context、Spring-context-suppo和Spring-expresssion（Spring表达式语言）等模块组成；

  • Spring-core模块：提供框架的基本组成部分，包括控制翻转（Inversion of Control,IoC）和依赖注入（DI）功能；
  • Spring-beans模块：提供了BeanFactory，是工厂模式的一个经典实现，Spring将对象 or 管理对象称为 Bean；
  • Spring-context模块：建立在Core和Beans模块的基础之上，提供一个框架式的对象访问方式，是访问定义和配置的任何对象的媒介，配置文件；
  • 其他框架省略

1.3 AOP 和 Instrumentation

  • Spring-aop模块：提供了一个符合AOP要求的面向切面的编程实现，允许定义方法拦截器和切入点，将代码按照功能进行分离；
  • Spring-aspects模块：提供了 AspectJ 的集成功能，AspectJ 是一个功能强大且成熟的AOP框架；
  • Spring-Instrumentation模块：提供了类植入支持和类加载器的实现；

1.4 消息

Spring 4.0 以后新增了消息模块，该模块提供了对消息传递体系结构和协议的支持

1.5 数据访问/集成

数据访问/集成层由 JDBC、ORM、OXM、JMS和事务模块组成；

1.6 Web模块

Web层由Spring-web、Spring-webmvc、Spring-websocket和Portlet模块组成；

1.7 测试模块

Spring-test模块支持使用JUnit或TestNG对Spring组件进行单元测试和集成测试。

2.1 创建一个Spring小程序

在项目里面创建一个类：

public class Springtest{
    public void sayHello(){
        System.out.println("Hello,Spring");
    }
}

Spring的核心是一个名为 applicationContext.xml 的配置文件，在src目录下创建该XML同名配置文件，内容会配置好：

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd http://www.springframework.org/schema/context https://www.springframework.org/schema/context/spring-context.xsd">

 <!-- 配置bean标签-->
 <bean id="spring_test" class="Springtest" />
</beans>

建立一个测试程序，在测试方法前面加上一个测试注解：@Test 标明这是一个测试方法，注意这种测试方法并不能接收键盘输入数据，只能在main方法接收键盘输入;

Spring是将创建对象的动作托管给Spring容器，那么我们在需要一个对象的时候就不需要显示的 new 操作符，只需要在配置文件配置Bean，那么我们就可以向容器索要Bean：

  1. 首先通过 ApplicationContext 接口实例从类路径目录中寻找指定的XML配置文件，就是上一步中创建的XML配置文件

  2. 然后通过容器来获取实例，调用getBean() 方法，传入XML文件中 bean 标签中的 id 字段值获取实例，getBean() 方法返回值是一个 Object 类型对象，所以最后需要强转。

    public Class test{
 @Test
 public void test_1(){
     // 1.指定XML配置文件位置
     ApplicationContext con = new           ClassPathXmlApplicationContext("applicationContext.xml");

     // 2.通过con实例来获取Bean
     //这里 getBean方法内的参数是XML配置文件中bean标签中id字段的值
     Springtest test = (Springtest)con.getBean("spring_test");
     test.sayHello();
 }
}

## 2.2 Spring IoC
Spring IoC(控制反转)是Spring框架的基础，也是Spring框架的核心理念，控制反转是一个比较抽象概念，**用来解决计算机程序的耦合问题**；

当人们需要一件东西时，第一反应就是找东西，例如在没有面包店两种情况下，您会怎么做？在没有面包店时，最直观的做法可能是您按照自己的口味制作面包，也就是一个面包需要主动制作。然而时至今日，各种网店、实体店盛行，已经没有必要自己制作面包。想吃面包了，去网店或实体店把自己的口味告诉店家，一会就可以吃到面包了。

注意，你并没有制作面包，而是由店家制作，但是完全符合您的口味。
上面只是列举了一个非常简单的例子，但包含了控制反转的思想，即把制作面包的主
动权交给店家。下面通过面向对象编程思想继续探讨这两个概念。

当某个Java对象（调用者）需要调用另一个Java对象时，调用者需要采用`new`操作符，这种方式会增加调用者与被调用者之间的耦合性，不利于后期代码的升级与维护；

当Spring框架出现后，对象的实例不再由调用者来创建，而是由Spring容器来创建，Spring容器会负责控制程序之间的关系，而不是由调用者的程序代码直接控制。这样，**控制权发生了反转**，这就是Spring框架的控制反转，控制反转的实现方法是 **依赖注入** 。

### 2.2.1 依赖注入的类型
依赖注入的作用是在使用Spring框架创建对象时动态地将其所依赖的对象（例如属性值）注入Bean组件中。依赖注入通常有两种方法，一种是使用构造方法，另一种是使用属性的setter方法注入。

#### 构造方法注入

使用属性的setter方法注入

使用 setter 方法注入是Spring框架中最主流的注入方式，利用JavaBean规范定义的setter方法来完成注入，灵活且可读性高；

public class StudentClass{
    student stu1;

    //setter方法
    public void setStu1(student stu1){
        this.stu1 = stu1;
    }

    public void dianMing(){
        System.out.println("班上的学生有："+stu1.getName()+
        ",家住："+stu1.getAddress());
    }
}

XML配置文件

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd http://www.springframework.org/schema/context https://www.springframework.org/schema/context/spring-context.xsd">
    <context:component-scan base-package="com"/>
    <bean id="student" class="student">
        <constructor-arg  name="name" value="建浩"/>
        <constructor-arg  name="address" value="广西灵山"/>
    </bean>

    <!--使用setter方法注入-->
    <bean id="stuClass" class="StudentClass">
        <!--这里的ref字段也可以使用为value来指定属性值-->
        <property name="stu1" ref="student" />
    </bean>
</beans>

上面就是通过setter方法的依赖注入，使用 bean 标签中的 property 字段来设置属性，注意 name 属性的值一定要是该类的成员变量名，不能出错。
